# S3 — Simple Storage Service

Gopherstack provides a full in-memory S3 implementation with versioning, multipart uploads, tagging, ACLs, presigned URLs, and Gzip-compressed object storage.

## Supported Operations

| Operation | Description |
|-----------|-------------|
| `CreateBucket` | Create a new bucket |
| `DeleteBucket` | Delete a bucket (must be empty) |
| `ListBuckets` | List all buckets in the account |
| `HeadBucket` | Check if a bucket exists |
| `GetBucketVersioning` | Get the versioning configuration |
| `PutBucketVersioning` | Enable or suspend versioning |
| `PutObject` | Upload an object |
| `GetObject` | Download an object |
| `HeadObject` | Get object metadata without body |
| `DeleteObject` | Delete an object or a specific version |
| `DeleteObjects` | Delete multiple objects in one request |
| `CopyObject` | Copy an object within or across buckets |
| `ListObjects` | List objects (v1 API) |
| `ListObjectsV2` | List objects (v2 API, recommended) |
| `ListObjectVersions` | List all versions of objects in a bucket |
| `PutObjectTagging` | Set tags on an object |
| `GetObjectTagging` | Get tags on an object |
| `DeleteObjectTagging` | Remove all tags from an object |
| `CreateMultipartUpload` | Initiate a multipart upload |
| `UploadPart` | Upload a part of a multipart upload |
| `CompleteMultipartUpload` | Finalize a multipart upload |
| `AbortMultipartUpload` | Cancel a multipart upload |
| `ListMultipartUploads` | List in-progress multipart uploads |
| `ListParts` | List parts of an in-progress multipart upload |
| `PresignedGetObject` | Generate a presigned URL to download an object |
| `PresignedPutObject` | Generate a presigned URL to upload an object |
| `GetBucketAcl` | Get the ACL for a bucket |
| `PutBucketAcl` | Set the ACL for a bucket |
| `PutBucketPolicy` | Set a bucket policy |

## AWS CLI Examples

```bash
# Create a bucket
aws --endpoint-url http://localhost:8000 s3api create-bucket --bucket my-bucket

# Upload an object
aws --endpoint-url http://localhost:8000 s3 cp file.txt s3://my-bucket/

# Download an object
aws --endpoint-url http://localhost:8000 s3 cp s3://my-bucket/file.txt ./file.txt

# List objects
aws --endpoint-url http://localhost:8000 s3 ls s3://my-bucket/

# Enable versioning
aws --endpoint-url http://localhost:8000 s3api put-bucket-versioning \
    --bucket my-bucket \
    --versioning-configuration Status=Enabled

# Upload and tag an object
aws --endpoint-url http://localhost:8000 s3api put-object \
    --bucket my-bucket \
    --key myfile.txt \
    --body file.txt \
    --tagging "env=dev&project=myapp"

# Delete multiple objects
aws --endpoint-url http://localhost:8000 s3api delete-objects \
    --bucket my-bucket \
    --delete '{"Objects":[{"Key":"file1.txt"},{"Key":"file2.txt"}]}'

# Multipart upload
UPLOAD_ID=$(aws --endpoint-url http://localhost:8000 s3api create-multipart-upload \
    --bucket my-bucket --key large-file.dat --query UploadId --output text)
aws --endpoint-url http://localhost:8000 s3api upload-part \
    --bucket my-bucket --key large-file.dat \
    --upload-id "$UPLOAD_ID" --part-number 1 --body part1.bin
aws --endpoint-url http://localhost:8000 s3api complete-multipart-upload \
    --bucket my-bucket --key large-file.dat --upload-id "$UPLOAD_ID" \
    --multipart-upload '{"Parts":[{"PartNumber":1,"ETag":"..."}]}'
```

## Known Limitations

- Bucket policies are stored but not enforced (no IAM policy evaluation).
- Server-side encryption (SSE-S3, SSE-KMS, SSE-C) fields are accepted but not applied; objects are stored unencrypted.
- S3 replication is not supported.
- S3 Object Lambda and S3 Access Points are not supported.
- `GetBucketLifecycle`, `PutBucketLifecycle`, `GetBucketCors`, `PutBucketCors` are not implemented.
- Cross-region bucket constraints are not enforced.

## Configuration

S3 runs on the main Gopherstack port (default `8000`). No separate configuration is required.
Objects are Gzip-compressed in memory to reduce heap usage. This is transparent to clients.
