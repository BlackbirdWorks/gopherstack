## Service UI Guidelines

When making a UI for a service, make sure:
- UI uses the service functions via the aws sdk
- The Service is a tab at the top
- The Service's operations are documented in Docs
- There is an e2e test using the UI fully for that service (covering all interactions)
- The Service has metrics displayed on the metrics tab
- Services are accessible via a tab at the top
- All services must look good in light and dark mode
- All services must define demo data to show in demo to visual inspections
